# Agent Guidelines for Agentuity Examples

## Repo Structure

Collection of standalone example projects organized by category:

| Category | Purpose |
|----------|---------|
| training/ | Progressive tutorials (01 through 05) |
| integrations/ | Frontend framework patterns (Next.js, TanStack, Turborepo) |
| frameworks/ | AI framework agents (Mastra, LangChain, OpenAI) |
| features/ | Platform capabilities (sandbox, cron, streams, port exposure) |

Each example is independently runnable. No shared dependencies between examples.

## Commands

From any example directory:

- `bun run dev` — start dev server (localhost:3500)
- `bun run build` — compile
- `bun run typecheck` — TypeScript checks
- `bun run deploy` — deploy to Agentuity cloud

## Project Conventions

Every example follows these patterns:

- **Runtime**: Bun + TypeScript (strict mode)
- **Schema**: `@agentuity/schema` (import as `s`) for agent schemas. Zod, ArkType, and other StandardSchema-compatible libraries also work.
- **Agents**: `createAgent()` with input/output schemas, default export
- **Routes**: `createRouter()` with Hono patterns
- **Frontend**: React + `useAPI` hook from `@agentuity/react`
- **Logging**: `ctx.logger`, never `console.log`
- **Styling**: Dark theme (#0a0a0a), cyan accent (#00FFFF), system-ui fonts
- **Path aliases**: `@agent/*`, `@api/*`, `@lib/*`

## Adding a New Example

1. Pick the right category directory
2. Run `agentuity create` inside it
3. Implement following the conventions above
4. Write a README matching the style of sibling examples
5. Update the category README table
6. Update the root README table

## Special Instructions

- Each example must be independently runnable (`bun install && bun dev`)
- Don't add type annotations to handler params — let TypeScript infer from schemas
- Don't create documentation files unless explicitly asked
- The `.gitignore` excludes per-example `AGENTS.md` files (auto-generated by CLI)
- Framework examples (frameworks/) don't have React frontends
- Training and features examples include React frontends

## CLI Reference

Each example has an auto-generated AGENTS.md (gitignored) with project-specific guidance. For detailed agent, route, and eval patterns, see the CLI's built-in docs at `node_modules/@agentuity/cli/AGENTS.md` inside any example.
