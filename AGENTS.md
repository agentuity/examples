# Agent Guidelines for Agentuity Examples

## Repo Structure

Example projects organized by intent:

| Folder | Overview |
|--------|----------|
| `training/` | Learn Agentuity step by step |
| `existing-apps/` | Add Agentuity agents to an existing app (e.g., Next.js, TanStack Start) |
| `integrations/` | Use Agentuity with another library, framework, or SDK |
| `features/` | See what the platform can do |

## Adding a New Example

1. Pick the right folder:
   - Adding agents to an existing frontend framework? → `existing-apps/`
   - Integrating with an external SDK/framework? → `integrations/`
   - Showcasing a platform capability? → `features/`
   - New training lesson? → `training/` (numbered, sequential)
2. Run `agentuity create` inside the chosen folder
3. Write a README matching the style of sibling examples
4. Update the folder's README table
5. Update the root README table

## Conventions

- **Logging**: `ctx.logger`, never `console.log`
- **Schema**: `@agentuity/schema` (import as `s`). Zod, ArkType, and other StandardSchema-compatible libraries also work.
- The `.gitignore` excludes per-example `AGENTS.md` files (auto-generated by CLI)

## CLI Reference

Each example has an auto-generated AGENTS.md (gitignored) with project-specific guidance. For detailed agent, route, and eval patterns, see the CLI's built-in docs at `node_modules/@agentuity/cli/AGENTS.md` inside any example.
